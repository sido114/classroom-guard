# Classroom Guard Specs

This directory contains feature specifications for incremental development.

## Spec Template
```markdown
# [Feature Name]

## Goal
One sentence describing what we're building.

## Acceptance Criteria
- [ ] Criterion 1 (testable)
- [ ] Criterion 2 (testable)
- [ ] Criterion 3 (testable)

## Technical Notes
- Implementation hints
- API endpoints
- Database changes
```

## Naming Convention
- Format: `NNN-feature-name.md` (e.g., `001-session-entity.md`)
- Status: Prefix with `[DONE]` when complete

## Usage
1. Create spec file with clear acceptance criteria
2. Tell Kiro: "Implement spec NNN"
3. Kiro implements, tests, and reports results
4. Review and iterate

## Tips
- Keep specs small (15-30 min each)
- One feature per spec
- Make criteria testable
